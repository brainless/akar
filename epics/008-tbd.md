## Cleanup Task: Rename `akar_layout::Rect` to `akar_layout::WorldRect`

**Goal:** Eliminate the naming ambiguity between `akar_layout::Rect` (world-space bounding box, introduced in Epic 004) and `taffy::geometry::Rect` (padding/margin layout type, brought into scope via `pub use taffy::prelude::*`). The ambiguity surfaces in `akar-layout/src/lib.rs` and any file that uses both types.

**Rename:** `Rect` → `WorldRect` everywhere it refers to the canvas world-space bounding box type.

**Files to update:**

| File | Change |
|---|---|
| `crates/akar-layout/src/rect.rs` | Struct definition: `pub struct Rect` → `pub struct WorldRect` |
| `crates/akar-layout/src/canvas_transform.rs` | `Rect` in function signatures and return types; `compute_visible_world_rect` return type |
| `crates/akar-layout/src/lib.rs` | `pub use rect::Rect` → `pub use rect::WorldRect` |
| `crates/akar-components/src/canvas.rs` | `CanvasResponse.visible_world_rect` type, `is_visible_world` arguments, `CanvasPainter::push_quad` argument, test `use` imports |
| `examples/canvas-basic-rust/src/main.rs` | `use akar_layout::Rect` import |
| `examples/demo-rust/src/main.rs` | `use akar_layout::Rect` import if present |

No logic changes — identifier rename only. `cargo clippy --workspace -- -D warnings` and `cargo test --workspace` must pass clean after the rename.
